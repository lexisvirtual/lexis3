# PROJECT VISION: Lexis Academy — O Ecossistema de Elite

Este documento é a **Fonte Única da Verdade (Source of Truth)** para a estratégia do projeto Lexis. Ele consolida a pedagogia, o marketing e a engenharia técnica em uma linha lógica única e inquebrável.

---

## 1. LINHA LÓGICA (THE CORE LINE)

A eficácia deste projeto reside na integração perfeita de três camadas:

1.  **Core Pedagógico (O "Quê"):** Inglês como Habilidade Motora. O idioma não é aprendido (teoria), é treinado (prática procedural).
2.  **Mecanismo de Marketing (O "Como"):** Narrativa de Inocência do Aluno + SEO Programático. Inocentamos o aluno pelo fracasso passado e atacamos o método tradicional, usando automação para dominar as buscas onde o Google é fraco.
3.  **Motor Técnico (O "Motor"):** Protocolo Leo + Roger AI + Knowledge System V9. Automação de ativos, auditoria de qualidade e otimização de SEO baseada em dados reais, sem intervenção humana constante.

---

## 2. ETAPAS DO RACIOCÍNIO (THE REASONING STEPS)

1.  **A Dor:** O mercado está cheio de alunos frustrados que "estudam há anos e não falam".
2.  **A Causa:** Metodologias tradicionais focam em gramática declarativa (conhecimento) em vez de memória procedural (habilidade).
3.  **A Solução Lexis:** Um treinamento intensivo (120h em 2 semanas) que trata o inglês como um esporte ou instrumento musical.
4.  **A Escala Digital:** Para atrair esses alunos, criamos um ecossistema de conteúdo que não apenas informa, mas doutrina o leitor no Axioma Lexis ("Idioma se treina").
5.  **A Eficiência Técnica:** O conteúdo não é escrito por humanos, mas por um sistema de IA auditado (Roger) que garante precisão cultural e metodológica, enquanto o motor SEO (Leo) garante que as páginas certas subam no ranking.

---

## 3. IDENTIFICAÇÃO DE RISCOS (RISK MAP)

| Risco | Impacto | Mitigação |
| :--- | :--- | :--- |
| **Alucinação da IA** | Conteúdo pedagogicamente errado ou culturalmente impreciso. | **Roger AI (Guardian):** Auditoria sistemática com threshold de qualidade. |
| **Dependência de APIs** | Quebra de imagens ou falha na geração de conteúdo (Pixabay/OpenAI). | **Knowledge System V9:** Fallback de imagens locais e cache de pautas. |
| **Over-SEO** | Conteúdo que parece "robótico" e afasta o usuário real. | **Axioma de Inocência:** Todo post deve obrigatoriamente seguir a narrativa humana de frustração vs. treino. |
| **Instabilidade de Build** | Erros de sincronia Git ou conflitos de pacotes (React 19). | **Deploy Instructions:** Uso de `--legacy-peer-deps` e scripts de automação robustos. |

---

## 4. ESTRATÉGIAS DE FORTALECIMENTO (STRENGTHENING)

*   **Audit Engine (Roger):** Evoluir o Roger para monitorar a "Consciência do Blog" via EMA (Média Móvel Exponencial). Se a qualidade cai, a publicação para.
*   **Leo Protocol v1.2:** Mudar de "publicação por volume" para "alocação de páginas". O Leo identifica onde o Google está fraco (Reddit, Quora) e ocupa o espaço.
*   **Knowledge System V9.1:** Self-hosting total de imagens. O sistema otimiza para WebP e hospeda localmente para garantir 100% de uptime visual.
*   **Interlink Programático:** O sistema cria uma malha semântica entre os posts automaticamente, aumentando a autoridade sem depender de plugins.

---

## 5. DESENVOLVIMENTO DOS TÓPICOS

### 5.1. Metodologia Lexis (O Axioma)
*   **Fases:** Start (Fonética), Run (Estruturas), Fly (Vocabulário), Liberty (Conversação).
*   **Princípio:** Redução da tradução mental através de alta densidade de prática. O professor é um treinador, não um palestrante.

### 5.2. Protocolo Leo (O Motor SEO)
*   **Fórmula de Prioridade:** `Score = (Impressões x 0.4) + (Posição Inversa x 0.3) + (Conversões x 0.3)`.
*   **Ações:** Upgrade de posts "quase ranqueando" (pos. 5-15) com FAQs e Snippets.

### 5.3. Roger AI (O Guardião)
*   **Função:** Auditar o IPL (Indicador de Performance Linguística) do conteúdo gerado.
*   **Upgrade Engine:** Posts com score < 60 são deletados; entre 60-84 sofrem upgrade estrutural; > 90 são considerados Elite.

### 5.4. Knowledge System V9 (Infraestrutura)
*   **Imagens:** Busca em cascata (Pixabay -> Fallback) com otimização via `wsrv.nl`.
*   **Arquitetura:** Markdown como fonte da verdade, GitHub Pages para velocidade e Cloudflare Workers para inteligência.
