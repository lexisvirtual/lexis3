# 🎯 VISÃO GERAL DO PROJETO: LEXIS ACADEMY

Este documento serve como a "Fonte Única da Verdade" para a estratégia, lógica e execução técnica do ecossistema Lexis.

---

## 1. 🧬 LINHA LÓGICA (THE GOLDEN THREAD)

A Lexis opera em um alinhamento perfeito entre três pilares fundamentais:

**PEDAGOGIA (O Produto)**
O inglês não é uma disciplina acadêmica, é uma **habilidade motora**. Treina-se como um esporte (Método 3F: *Phrase, Fluidity, Function*).

**MARKETING (A Narrativa)**
A "Narrativa da Inocência": O aluno não fracassou; o método tradicional é que falhou com ele. O SEO não é "blogging", é **alocação de páginas** em lacunas de autoridade do Google.

**TÉCNICO (O Motor)**
Automação total via **Leo Protocol** (SEO), **Roger** (Auditoria de IA) e **Knowledge System V9** (Ativos), garantindo escala com custo marginal zero.

---

## 2. 🧠 ETAPAS DO RACIOCÍNIO (THE LOGIC STEPS)

1.  **A Falha Sistêmica:** Identificamos que o ensino tradicional foca em "conhecimento declarativo" (saber sobre a língua), enquanto a fluência exige "conhecimento procedural" (saber fazer/falar).
2.  **A Solução por Treino:** Criamos uma imersão de 120h (2 semanas) que simula anos de exposição, focando em repetição deliberada e correção em tempo real.
3.  **A Escala de Autoridade:** Para atrair alunos, o sistema técnico deve ocupar o Google com conteúdo que respira essa metodologia, atacando a dor do "estudei e não falo".
4.  **A Autonomia de Ativos:** O projeto não pode depender de editores humanos. A IA deve criar, auditar, linkar e otimizar os conteúdos baseada em dados reais de performance (GSC).

---

## 3. ⚠️ IDENTIFICAÇÃO DE RISCOS (RISK ASSESSMENT)

| Risco | Impacto | Descrição |
| :--- | :--- | :--- |
| **Alucinação Cultural** | Alto | A IA pode sugerir contextos culturais errados (ex: festas brasileiras acontecendo em inglês). |
| **Dependência de APIs** | Médio | Quedas ou mudanças de preço no Pixabay/Gemini podem interromper o fluxo de publicação. |
| **Mudanças de Algoritmo** | Alto | Atualizações do Google (HCU) podem penalizar conteúdo gerado por IA se não houver "Helpful Content". |
| **Concorrência de Commits** | Baixo | Conflitos entre o Worker e edições manuais no repositório GitHub. |

---

## 4. 💪 ESTRATÉGIAS DE FORTALECIMENTO (STRENGTHENING)

-   **Roger (AI Content Guardian):** Sistema de auditoria com EMA (*Exponential Moving Average*). Ele deleta ou faz upgrade de posts baseando-se em um score de qualidade, garantindo que o nível médio do blog suba continuamente.
-   **Knowledge System V9:** Migração para imagens *self-hosted*. O sistema baixa, otimiza para WebP via `wsrv.nl` e commita no repo, eliminando links quebrados.
-   **Leo Protocol:** Otimização baseada em ROI. O sistema prioriza atualizar posts que estão na "página 2" do Google (pos. 5-15) para transformá-los em resultados de topo.
-   **Hibridismo Humano-IA:** Uso de bancos de dados curados (fallback images) e prompts com "Regras de Ouro" (Anti-Alucinação).

## 5. 📚 DESENVOLVIMENTO DOS TÓPICOS

### 5.1 Metodologia Lexis (O Core)
-   **Fases:** Start (Fonética) -> Run (Estruturas) -> Fly (Vocabulário) -> Liberty (Conversação).
-   **Princípio:** Idioma não se aprende, se treina. O erro é uma ferramenta de ajuste motor.

### 5.2 Leo Protocol (O Cérebro SEO)
-   **Fórmula de Score:** `(Impressões x 0.4) + (Posição Inversa x 0.3) + (Conversões x 0.3)`.
-   **Ações Automáticas:** Inserção de FAQ, otimização de snippets e interlinkagem automática por clusters semânticos.

### 5.3 Roger AI Guardian (A Auditoria)
-   **Upgrade Engine:** Divide os posts em níveis (Lixo, Estrutural, SEO, Autoridade, Elite).
-   **Auto-Healing:** Detecta posts com perda de performance e aciona uma reescrita parcial com modelos superiores (Llama 70b / GPT-4o).

### 5.4 Knowledge System V9 (A Infraestrutura)
-   **Arquitetura:** GitHub Pages (Host Estático) + Cloudflare Workers (Lógica de Borda) + Markdown (Source of Truth).
-   **Otimização:** Imagens redimensionadas para 1200px, 80% de qualidade, formato WebP.
